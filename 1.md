# 🧠 AI Investment System: Architecture & Implementation Blueprint

## 🎯 1. System Overview
This document outlines the architecture for an AI-powered Investment System focused on the Indian equities market. 

**Core Paradigm:** We have transitioned from a static rule-based system and a multi-agent "committee" to a **Single Agent + Tools (ReAct) Architecture**. 
* The system relies on deterministic Python functions (Services) for math and data gathering.
* A single, powerful LLM acts as the "Lead Portfolio Manager", utilizing LangChain's tool-calling capabilities to fetch data, assess risk, and generate a final investment thesis.

## 🛠️ 2. Tech Stack
* **Orchestration:** LangChain (`create_tool_calling_agent` / `AgentExecutor`)
* **LLM Engine:** Groq API (Llama-3-70B for fast, cheap, and precise tool calling)
* **Backend Framework:** FastAPI (Python)
* **Database (Relational & Memory):** Supabase (PostgreSQL) for user profiles, portfolio states, and saving historical AI investment theses.
* **Database (Vector / RAG):** Supabase (pgvector) for storing financial rules and macro-economic context.
* **Market Data:** `yfinance` or Indian brokerage APIs (Upstox/AngelOne).

⚙️ 4. Execution Flow (The ReAct Loop)
When an endpoint is triggered (e.g., POST /api/v1/analyze), the LangChain AgentExecutor begins a Reason + Act loop.

Example Flow for an Indian Stock (TATAMOTORS.NS):

Thought: The agent realizes it needs user context first.

Action: Calls fetch_portfolio(user_id="xyz").

Observation: Returns that the user has a 40% overexposure to the Auto sector.

Thought: The agent needs fundamental data to see if the stock is cheap enough to justify the risk.

Action: Calls fetch_quant_scores(ticker="TATAMOTORS.NS").

Observation: Returns PE: 12 (cheap), RSI: 28 (oversold).

Thought: The agent checks the knowledge base for sector-specific macro limits.

Action: Calls search_knowledge_base(query="Auto sector limits India").

Observation: Returns the rule: "Never exceed 35% sector allocation."

Final Verdict: The agent breaks the loop and outputs a structured JSON response.

📜 5. Output Schema

JSON
{

🤖 6. Instructions for AI Coding Assistant (Antigravity)
When modifying or extending this codebase, adhere strictly to the following rules:

No Math in the LLM: Never prompt the LLM to calculate averages, ROE, or technical indicators. All calculations must happen in the services/ directory, which are then exposed to the LLM via tools/.

Tool Docstrings are Critical: When writing tools in tools/, you must provide exhaustive, descriptive docstrings. The LangChain agent relies entirely on these docstrings to understand when and how to use the tool.

Single Agent Focus: Do not create multiple agent files (no quant_agent.py or risk_agent.py). Funnel all reasoning through lead_pm.py using the tool-calling loop.

Stateful Memory: Always ensure the generated investment_thesis is passed to db/thesis_repo.py so the system maintains memory of its past decisions for a specific user.
